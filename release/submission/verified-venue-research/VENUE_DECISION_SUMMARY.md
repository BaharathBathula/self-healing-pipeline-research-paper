# Venue Decision Summary

## Current Paper State

The current paper is a clean IEEE-format 8-page submission package with PDF, LaTeX source, BibTeX, figures, artifact statement, reproducibility checklist, manifest, and checksums.

## Verified Venue Ranking

### 1. IEEE Access

Decision: Best near-term journal target if APC is acceptable.

Why:

- Broad IEEE engineering scope
- Accepts applied engineering work
- Allows supplemental code/data material
- Fast publication path compared with traditional journals
- Current paper is already close to IEEE-style formatting

Main blocker:

- Must convert to IEEE Access template and likely expand from conference-style 8 pages into a stronger journal article.
- APC is high.

### 2. Journal of Big Data

Decision: Good target after expansion.

Why:

- Stronger thematic fit for data-intensive systems and data reliability
- Springer Nature venue with clear data availability expectations

Main blocker:

- Current manuscript is probably too short and too IEEE-conference-like.
- APC is high.
- Evaluation needs more depth.

### 3. SoftwareX

Decision: Backup target if repositioned as a software artifact paper.

Why:

- Artifact-focused venue
- Good fit if the repo is polished, versioned, documented, and easy to run

Main blocker:

- Current paper is not written as a software article.
- Needs artifact-first framing.

### 4. Future Generation Computer Systems

Decision: Stretch target, not recommended as first submission.

Why:

- Strong systems and big-data infrastructure relevance

Main blocker:

- Higher novelty/evaluation bar.
- Current paper likely needs major expansion and stronger baselines.

## Practical Recommendation

Do not submit blindly today.

Best path:

1. Choose IEEE Access as the primary near-term target only if APC is acceptable.
2. Convert the paper to the IEEE Access template.
3. Expand Related Work, Evaluation, Limitations, and Artifact Availability.
4. Keep Journal of Big Data as the second option.
5. Keep SoftwareX only if the implementation artifact becomes the main contribution.
6. Do not target FGCS until the paper has a much stronger systems evaluation.

## Final Day 27 Decision

Primary target candidate: IEEE Access

Backup target candidate: Journal of Big Data

Artifact/software target candidate: SoftwareX

Stretch target: Future Generation Computer Systems
