# Submission Risk Checklist

## Scope Risk

- [ ] Venue clearly accepts data engineering / data reliability / pipeline automation work
- [ ] Paper is not too applied for the venue
- [ ] Paper is not too systems-oriented for a data journal
- [ ] Paper is not too short for a journal article
- [ ] Paper is not too long for a conference submission

## Format Risk

- [ ] Template matches target venue
- [ ] Page limit verified
- [ ] Reference format verified
- [ ] Figure/table placement acceptable
- [ ] Supplementary files allowed

## Novelty Risk

- [ ] Contribution is clearly positioned beyond monitoring-only systems
- [ ] Automated recovery contribution is explicit
- [ ] Evaluation claims are not overstated
- [ ] Synthetic data limitation is clearly acknowledged
- [ ] Baselines and comparison framing are defensible

## Compliance Risk

- [ ] Author name confirmed
- [ ] Author email confirmed
- [ ] Affiliation confirmed
- [ ] Conflict of interest statement ready
- [ ] Funding statement ready
- [ ] Data availability statement ready
- [ ] AI-assistance disclosure checked against venue policy

## Artifact Risk

- [ ] ZIP opens correctly
- [ ] PDF opens correctly
- [ ] LaTeX source included if required
- [ ] BibTeX included if required
- [ ] Figures included
- [ ] Reproducibility checklist included
- [ ] Artifact statement included
- [ ] No temporary build files included

## Decision

- [ ] Submit to this venue
- [ ] Hold and revise
- [ ] Use as backup
- [ ] Reject venue
